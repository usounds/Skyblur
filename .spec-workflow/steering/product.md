# Product Overview

## Product Purpose

Skyblur は、Bluesky に投稿する文章の一部を伏せ字として公開し、Skyblur 上で条件を満たした閲覧者だけが伏せられた本文や補足情報を読めるようにするアプリケーションである。

Bluesky のタイムライン上ではネタバレやセンシティブな情報を不用意に見せず、読みたい人だけが Skyblur の投稿ページで続きを確認できる体験を提供する。AT Protocol と Bluesky の既存機能を活用しながら、投稿者が「誰にどこまで見せるか」を柔軟に制御できることを重視する。

## Target Users

- **Bluesky にネタバレを投稿したいユーザー**: 映画、漫画、ゲーム、イベントなどの内容を話したいが、タイムライン上で未読・未視聴の人に直接見せたくない。
- **限定的に情報を共有したいユーザー**: ログイン済みユーザー、フォロワー、相互フォロー、特定条件を満たす相手だけに補足情報を見せたい。
- **Skyblur 投稿を読むユーザー**: Bluesky から Skyblur のリンクに移動し、自分に閲覧権限がある場合に伏せ字の内容を確認したい。
- **AT Protocol 互換クライアント利用者**: Bluesky の投稿やレコードを扱うクライアントから、Skyblur の投稿・閲覧機能を自然に利用したい。

## Key Features

1. **伏せ字投稿**: 投稿本文の一部を伏せ字として Bluesky に投稿し、Skyblur 上で元の文章を表示する。
2. **補足情報**: Bluesky には表示しない追加情報を Skyblur 上でのみ提供する。
3. **公開範囲制御**: 全体公開、ログイン必須、パスワード必須、フォロワー限定、フォロー中限定、相互フォロー限定などの条件で閲覧を制御する。
4. **Bluesky/AT Protocol 連携**: OAuth、XRPC、Lexicon、AT URI を通じて Bluesky アカウントと投稿データに連携する。
5. **投稿管理**: 自分の Skyblur 投稿の作成、編集、削除、一覧表示、投稿詳細表示を提供する。

## Business Objectives

- Bluesky ユーザーがネタバレや限定情報を安心して投稿できる場を提供する。
- Skyblur の投稿リンクを Bluesky 上で自然に共有できる体験を維持する。
- AT Protocol の仕様に沿った拡張可能な Lexicon と API を保ち、将来のクライアント連携を容易にする。
- 閲覧制御の信頼性を高め、投稿者と閲覧者の期待を裏切らないアクセス制御を実現する。

## Success Metrics

- **投稿成功率**: 投稿作成・更新フローがエラーなく完了する割合。
- **閲覧成功率**: 権限を持つ閲覧者が Skyblur 投稿を正常に表示できる割合。
- **認可の正確性**: 権限のない閲覧者に伏せ字本文や補足情報を漏らさないこと。
- **回帰の少なさ**: 既存の公開範囲、OAuth、XRPC、Lexicon 生成、E2E テストに重大な回帰がないこと。
- **UI の理解しやすさ**: 投稿者が公開範囲を誤って選ばないよう、ラベル・説明・エラー表示が明確であること。

## Product Principles

1. **伏せることを第一級の体験にする**: Bluesky 上で見せる情報と Skyblur 上で見せる情報の違いを、投稿者にも閲覧者にも分かりやすくする。
2. **アクセス制御は deny-by-default にする**: 認証、認可、外部 API、保存データのいずれかが不確かな場合、非公開コンテンツは表示しない。
3. **既存の AT Protocol 文化に寄り添う**: DID、AT URI、Lexicon、XRPC、Bluesky の関係性やリストなど、既存概念を尊重して設計する。
4. **投稿者の意図を守る**: 公開範囲の選択、パスワード、関係性、リストなどの条件を一貫して解釈し、UI と API の挙動を一致させる。
5. **小さく安全に拡張する**: 新機能は既存の制限付き投稿の仕組みを再利用し、必要な箇所に限定して変更する。

## Monitoring & Visibility

- **Dashboard Type**: Web アプリ UI、開発時の spec-workflow dashboard、テストレポート。
- **Real-time Updates**: 投稿作成・認証・閲覧判定は UI のローディング状態と通知で進行状況を示す。
- **Key Metrics Displayed**: ユーザー向けには投稿成功、閲覧可否、ログイン必須、権限不足、パスワード要求を明確に表示する。
- **Sharing Capabilities**: Bluesky 投稿、Skyblur 投稿 URL、ユーザープロフィール、カスタムフィードを通じて共有する。

## Future Vision

Skyblur は、Bluesky 上で直接見せすぎない投稿体験を支える、AT Protocol ネイティブな伏せ字・限定公開ツールとして発展する。公開範囲の選択肢を増やす場合も、投稿者が理解しやすく、閲覧者が納得でき、サーバー側で安全に検証できることを優先する。

### Potential Enhancements

- **リスト限定公開**: 投稿者自身の Bluesky リストを使って、より細かな閲覧制御を提供する。
- **公開範囲の説明強化**: 投稿前に誰が閲覧できるかをより明確にプレビューする。
- **クライアント連携**: Lexicon と XRPC を整備し、Skyblur 以外の AT Protocol クライアントからも扱いやすくする。
- **信頼性向上**: OAuth、XRPC、外部 API 依存部分のタイムアウト、診断、テストを継続的に強化する。
